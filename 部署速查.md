## ⭐ 3 条铁律（最容易忘）

1. **所有 `fly` 命令前，先进项目文件夹**（fly 靠文件夹里的 fly.toml 认 app）：
   ```
   cd "/Users/yuqiaohuang/MyProjects/AI Commander"
   ```
2. **`fly deploy` 部署的是「你电脑本地的文件」，不是 GitHub。** `git push` 只是备份，**不会部署**。想更新线上，必须跑 `fly deploy`。
3. **网址永远不变**，改了版本也不用换链接。

## 🔄 改完代码、更新线上版本

**最简单（直接在主文件夹改）：**
```
cd "/Users/yuqiaohuang/MyProjects/AI Commander"
# ...改代码...
git add -A
git commit -m "改了啥"
git push origin main      # 备份到 GitHub（可选，不影响部署）
fly deploy                # ← 这一步才是真正部署，等 3-5 分钟
```
跑完显示 `successfully deployed` 就成，刷新网址就是新版本。

**如果你是在 worktree 里改的（多一步「合并进主文件夹」）：**
```
# worktree 里 git commit 之后：
cd "/Users/yuqiaohuang/MyProjects/AI Commander"
git merge 你的worktree分支名     # 把改动搬进主文件夹（关键！）
fly deploy
```
