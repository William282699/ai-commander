# 改动方向（傻瓜版）— 把游戏改成「全程跟参谋对话」

> 这份只讲**改什么方向**，以及**游戏会从什么样变成什么样**。不写怎么实现。
> 目的：先确认方向对不对，确认了再写具体实施方案。
> （技术细节版另见 `PLAYTEST_FEEDBACK_AND_FIXES.md`，那份是给 Codex 落地用的。）

---

## 一句话目标

让玩家**全程只是在跟参谋说话**：说了就办，重要的事参谋主动跟你说——找回「当司令」的感觉。

---

## 这次要改的 4 件事（其实是同一件事的四个面）

### 改动 1：拿掉 A/B/C 选择卡

- **现在：** 你说「派 Carter 去北线」→ 弹出一张卡：A 正面推进 / B 侧翼包抄 / C 先侦察，你得点一个。
- **改成：** 你说「派 Carter 去北线」→ Chen 直接回「Carter 北上，3 分钟到位。」部队就动了。
- **感觉差别：** 从「做选择题的人」→ 变回「下命令的司令」。
- **注：** 这一刀有**前端 + prompt（给 AI 的指令）两半**——前端负责不弹卡，prompt 负责让 AI 别再生成一堆选项。prompt 具体改哪、怎么分两步走，见下面「给 AI 的指令要不要改」专门一节。

### 改动 2：Emily 别动不动就让你确认

- **现在：** 你说「造 3 辆主战坦克」→ Emily 也弹卡让你确认。明明已经说得很清楚了。
- **改成：** 你说「造 3 辆主战坦克」→ Emily 回「3 辆主战坦克，排进生产队列了。」钱够就直接造；钱不够才问「只够造 1 辆，先造着？」
- **感觉差别：** 清楚的命令不再被反问。

### 改动 3：把「战报」和「参谋说话」分开

- **现在：** Chen 的对话框里混着「北线受到攻击」「弹药 40%」这种系统播报——看着像 Chen 在念警报机器人，也分不清哪条重要、哪条是背景音。
- **改成：** 系统战报（「北线接触敌军」「某据点失守」）进**旁边一条单独的、安静的「战报」道**；Chen / Marcus / Emily 的对话框里**只有他们作为人说的话**。
- **感觉差别：** 一来不吵了；二来 Chen 只说人话，更像个真人，而不是广播喇叭。
  （前线战报本来就不是将军亲口播报的，把它挪出去才对。）

### 改动 4：Chen 自己扛小事，只把大事丢给你

- **现在：** 要么弹卡，要么每件事都等你亲自下令。
- **改成：**
  - **明显该做又安全的事**（拿闲着的预备队去补一条快塌的线）→ Chen 自己办了，跟你说一声：「南线告急，我把 Carter 调过去了，能守住。」
  - **真正两难的事**（救北线就得抽空中路）→ Chen 不自己动，问你一句：「北线要塌，抽中路的 Carter，还是让北线先顶？」你一句话回。
  - **真听不懂的**（「北边」指哪）→ Chen 问一句：「北边是指北线还是火车站？」
- **感觉差别：** 你手下像有个能干的参谋——小事他扛了，大事他问你。**这就是「司令感」。**
- **顺带:** 系统触发的「紧急卡」(据点要丢、HQ 告急时弹的那张)**也一起拿掉**,走这同一套——能自动救就自动救 + 回报,要付代价就问你。**但有个不对称要记住:自动增援很安全(加兵几乎不会错),自动撤退很危险(可能违背你"死守"的意图)——所以撤退要看你设的 doctrine,没设就问你,别自作主张撤。** 真·存亡时刻(HQ 要没)不弹卡,改成"最大声的 Chen + 已经在动手",该揪你注意力照样揪。

### 改动 5：打下 / 丢掉据点，要真的有得有失

- **现在：** 占据点、丢据点基本只是个胜负计数——Chen 说「我们丢了南线」，你心里没什么波澜，因为丢了也不疼。
- **改成：** 占下一个据点 → 持续给你钱/油（按据点位置和类型，越重要给得越多）；丢了 → 这份收入就没了。
- **感觉差别：** Chen 那句「南线告急，我把 Carter 调过去了」终于**有分量**了——因为南线真的值钱，守住和丢掉是两个结果。这是给前面 4 件事「兜底的赌注」。
- **⚠️ 怎么做：** 这件**单独做、单独测**，别和砍卡那几件揉在一起。其中**「占了给收入」是必做的便宜大头**；**「高地加攻击力」这类地形加成是锦上添花，时间紧可以先放**。

## 整体：游戏从什么样 → 变成什么样

**改之前：**
你打字/说话下令 → 弹卡让你选 A/B/C → 聊天框里参谋对话和系统警报糊成一团 → 你忙着读、忙着点 → 像在填表，没有当司令的感觉。

**改之后：**
你说话 → 事就办了，Chen 回你一句 → 重要战报在旁边一条道安静地刷，扫一眼就懂 → Chen 偶尔自己拿了主意跟你说一声，或者把一个真正的难题摆到你面前让你拍板 → 你感觉手下有一队活人参谋，而你是那个发号施令、做关键决断的司令。

## 给 AI 的指令（prompt）要不要改？改哪里

**要改，但分两步走，而且大部分别碰。**

先记住一个分法：AI 干两种**完全不同的活**，所以可能要两套指令——

- **活 A：听懂你下的命令。** 用现在那个大 prompt。→ **改它，不重写。**
- **活 B：替引擎用 Chen 的口气说话**（自己调兵后回报、或把难题问你）。→ 这是个**新的小指令**。

### 活 A（听懂命令）的大 prompt —— 只动这几句

- **第 104 行**：现在写「每次都给司令 1-3 个方案」→ 改成「命令清楚时**只给一个方案、直接办**；只有你明确说『给我几个选择』才给多个」。**这一句就是 A/B/C 卡的源头。**
- **第 151 行**：现在写「下命令 → 执行，返回 1-3 个选项」→ 改成「下命令 → 执行，返回一个方案 + 用 Chen 的口气报一句」。
- **加一条**：「有风险 → 用 CONFIRM：Chen 顶你一句（『确定？南线会空』），你说行了再办；真听不懂 → 用 ASK 问一句；**能猜出来的就别问，直接按最合理的办**。」（CONFIRM / ASK 这些词 prompt 里本来就有，不用发明。）
- **千万别碰**：中间那一大段——怎么理解中英文命令、否定、地名匹配、Chen 的人设——**一个字别动**。那是你哥验证「听得懂」的命根子。
- `risk` / `reward` / `recommended` 那几个字段**留着别删**（它们只是没人用的剩饭，删了反而可能碰坏校验）。

### 活 B（替引擎说话）的新小指令

- 喂给它的**不是**完整战场，是**引擎已经算好的几条事实 + 引擎的决定**（「南线要塌、Carter 决定性增援、可自己动」）。
- 它的活就一句：「你是 Chen，把这事用 1-2 句你的口气说出来。」
- **它不决定该不该动兵——引擎已经定了，它只负责说。**

### 一条铁律（别让 Codex 搞反）

**决策永远在引擎（算账 + 死规矩），LLM 两种活都只管「翻译 / 开口」，不拍板动兵。** 别做成「每隔几秒把战场喂给 LLM、让它自己决定要不要动手」——那又贵、又慢、又不稳。等以后本地推理便宜了再说。

### 两步走（保住已经能跑的）

1. **第一步：prompt 一个字不改，光改前端**——让它自动执行推荐的那个方案、不弹卡。解析没动，「听得懂」就不会坏。
2. **第二步（以后优化）：再动上面 prompt 那几句**，让它别白生成 2 个用不上的方案（省延迟、省钱）。

## 本轮怎么排

- **改动 1-4 是同一件事，捆在一起做**：让所有信息各归各位——**事实进战报道，人话进对话框，命令说了就办。** 只改其中一件都不完整（比如光砍卡、不分战报，对话还是吵）。
- **改动 5（据点有分量）是本轮第二步，单独做、单独测**：它给前面那套对话兜上「赌注」，让 Chen 的回报有分量。建议在改动 4 前后落地，这样「Chen 救了南线」才真的让人有反应。

## 下一轮（已经排期，不会忘）

- **Chen 越玩越懂你（校准种子）**：把你每次拍板的选择记下来（比如「这司令总保北线、不爱动预备队」），下次 Chen 就照你的风格来，还冒一句「Chen 记住了：…」让你看见他在适应你。
  - **为什么排在改动 4 之后：** 它记的就是改动 4 里你做的那些决定——没有「Chen 把难题摆给你、你拍板」这个动作，就没东西可记。所以它天然接在改动 4 屁股后面。
  - 这是你长期愿景（AI 越来越懂你）的**第一个真种子**，所以一定做，只是排在这轮之后。
- **敌人改「节奏 + 看得懂」**：让进攻早点来、让 Chen 把敌人动向解读给你听、加「找你防线缝」的地图分析。这是「让你有仗可指挥」，另起一轮。

## 校准种子：现在先「干净地记 input」，以后再建个性化

> 这一节回答「个性化 AI commander 怎么起步」。核心:**现在不建系统,只干净地攒数据。**

**原则:数据是唯一不可逆的东西。** 算法以后随时能写;但这个月真玩家的反应,**没记下来就永远没了**。所以现在最该做、风险最低的一步,是**把入口数据干净地记下来**。

**为什么不碰架构:** 记录就是「在旁边看着、往别处写一笔」——**不改任何决策、流程、prompt**。它是观察,不是修改,天生破坏不了已验证的东西。

**每个校准时刻记一条,内容:**
- **当时那几个特征**(引擎做决策本就用的:哪条线、哪支队、什么动作、是不是预备队)——引擎手里现成的。
- **Chen 做了什么**(自己增援南线 / 撤退 / 问你)。
- **玩家原话反应**(「你在干嘛」)——**记原话,别只记「反对」这个标签**(标签会过时;原话里藏着"他气哪个点",以后要重读)。
- **确认那一来一回**(Chen 提议的规则 + 你的「对/不是」)。
- **把反应和它对应的那个动作连起来**(时间戳 / 动作 id),否则以后配不上对。
- **谁**(哪个 session / 玩家),为了以后跨局拼出"这个人的档案"。

**怎么记:** 记在**服务器**(能收集所有测试者的,不是散在各人浏览器);**事件驱动**(只在 Chen 自主动作、玩家反应、确认这几个时刻记,不是每帧狂记);**宁可多记原始的、轻结构**(以后能删字段,补不回没记的)。

**一鱼两吃:** 这套记录正好**也解决了「他们到底玩没玩」的睁眼瞎**——当下能看测试者干了啥,将来是个性化的干净原料。

**以后才建的(不是现在):** 用攒下来的数据,从真反应里长出「偏好维度表」、调引擎闸门,做真正的个性化。现在只管干净地攒。

(隐私:等陌生人来测时,对「会记录你的指令」透明一点;亲友阶段无所谓。)
